JavaScript中的undefined

发表于 2012-03-30   |  

JavaScript的undefined是个非常有趣的东西。

  1. 首先，undefined在JavaScript里面是一种原始类型，并且类型有且仅有一个值，就是undefined；
  2. JavaScript在全局范围定义了一个变量，这个变量的值是undefined，他的名字也是`undefined。

undefined的不确定性

但是这个全局变量并非一个常量，也不是一个保留字。也就是说，这个值是可以被更改，所以这个语句完全是有效的：

undefined = "something else";

由于undefined的不确定性，所以需要判断一个变量是否是undefined的时候不能直接使用如下语句：

if (o.abc === undefined) {
    // do something
}

使用技巧

常见的更为可靠的方法有：使用typeof操作符、void操作符、函数形参。
typeof操作符返回操作数的类型（字符串），若操作数是undefined类型，那么返回”undefined”，使用方法如下：

typeof variable === "undefined";

void操作符执行表达式并返回undefined：

function isUndefined(obj) {
    return obj === void 0;
}

在JavaScript中，当函数的形参的个数多于实际传入的参数时，多余部分的形参的值将设成undefined。故可通过此方法来确定undefined。

(functon abc(root, undefined) {
    if (root.abc === undefined) {
        // do something
    }
}(this))
